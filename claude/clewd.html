                                                        # replit部署

                                                        fork项目地址open in new window

                                                        点击tools栏中的secrets，移动端滑到最右侧点击Tools栏中的secrets

                                                        点击new secret 添加CookieArray,值为你前面抓取的cookie 添加username和password`（可选）两个key自行设置[1]

                                                        点击run，之后在页面中点击复制反代地址

                                                        # codesandbox部署

                                                        前言

                                                        具体注册步骤和更多的教程可以看原版教程，由江贤之撰写：https://sqivg8d05rm.feishu.cn/wiki/Y7R0wBdFMixvPMk8YRJcQJ8unlb

                                                        1. 打开官网open in new window进行注册
                                                        2. 登录之后，选择import repository， 选择授权all repository
                                                        3. 之后在github repository URL中填入clewd的仓库地址（如果你有fork的话直接填你仓库的地址，省去sandbox帮你fork的那一步）
                                                        4. 创建之后,在右边会有一个窗口用于初始化项目，只需要一路next到最后重启项目就行
                                                        5. 由于codesandbox限制不能创建config.js文件,所以我们使用环境变量来填写cookie和其他的变量；点左上角的logo，往下找到setting，然后找到ENV variables,然后参考上面的replit部署的secrets填写方法即可

                                                        key填写CookieArray,值为cookie，然后点击add添加,接着按需求填写密码和用户名，添加之后点击重启

                                                        多cookie

                                                        如果有多个cookie可以中间,隔开，如：cookie1,cookie2,cookie3

                                                        关于ip

                                                        codesandbox的虚拟机有一层cloudflare的cdn，不确定ip是否会变更，等待后续更新，若429需要更换ip,看下面解决方法

                                                        # 接入酒馆

                                                        若使用的是破限组的replit酒馆，请参考https://sqivg8d05rm.feishu.cn/wiki/E925wr6MKixS44kJUBtcB0uYnQe#Ce5QdRdqao109Cx5d2kcSi79neg, 本文使用的是原版的傻酒馆

                                                        先打开api接口，查看类型是否为openai，同时打开Show "External" models (provided by API)

                                                        之后转到第一个选项，往下找到openai反向代理，填入你刚刚复制的clewd代理链接即可，如果有密码那些记得也一起填了

                                                        完事了

                                                        # 429方法

                                                        可以通过提交后创建新分支方法解决，效果不一定好[2]

                                                        点击右上角的commit and push旁边的右边三角形展开更多选项，选择create branch form master 然后就会创建一个新分支，理论上环境变量在一个项目中通用，若消失了重复上面的步骤重新填一遍即可

                                                        1. 此举措是为了防止有人使用脚本盗取反代地址，可自行选择是否设置，值分别代表着用户名和密码，可以自行设置 ↩︎

                                                        2. 鸦鸦注：还有一种可能就是重启太多次或者到时间了，可以选择等一下或者更换/添加更多cookie ↩︎
